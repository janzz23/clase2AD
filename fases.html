Fase Actividades
Iniciacion
  • Nace la idea del producto de SW
  • Se Identifican los clientes y segmentos del mercado
  • Se entrevistan algunos clientes para identificar sus necesidades y conocer de forma general las funcionalidades que deberia tener el SW
  • Se hace un análisis para determinar si existen otros productos similares
  • Se analiza la factibilidad del producto y la posible aceptación por parte de los usuarios.
Planeacion
  • se desarrolla un plan para determinar los recursos necesarios para implementar el proyecto de SW (Tiempos, cronograma, personal, etc)
  • Se determina el costo aproximado del proyecto y el tiempo de desarrollo.
  • Dependiendo de la fecha de lanzamiento del producto se puede ajustuar el cronograma.
  • El plan de desarrollo del producto se actualiza de acuerdo con las necesidades del proyecto (ej: si se incluyen nevas funcionalidades)
  • se desarrolla el Plan De Gestion De La Configuracion: contiene actividades para controlar los cambios y manejo de versiones de su proyecto de SW
Analisis
  • Se recopila la mayor cantidad de informacion posible sobre las necesidades de los clientes.
  • Se recopila informacion detallada sobre las funcionalidades del producto,, desempeño requerido para el producto, confiabilidad, seguridad, usabilidad.
  • Los requerimientos generales se pueden recopilar por medio de entrevistas y lluvias de ideas con los clientes.
  • Los requerimientos definen que es lo que el producto de SW debe hacer. hay requerimientos Funcionales y No Funcionales; funciones escenciales y caracteristicas del SW
Diseño
  • Esta se define como el SW, debe ser construido para cumplir con los requerimientos definidos en la fase anterior.
  • Se define la arqutectura del SW (divisiones en un sub-sistemas) y se diseña de forma especifica de cada componente del SW. Todo esto se acompaña de la respectiva documentación que es lo que conforma el documento de diseño del SW.
  • AUN NO SE DESARROLLA CODIGO
Implementación
  • En esta fase se programa todo el software
  • El Objetivo es tomar el diseño obtenido en la fase anterior y programarlo completamente usando las tecnologias que se hayan definido.
Pruebas
  • El SW, se prueba para determinar si cimple con los requerimientos
  • El Software se prueba en tres fases:
  • Cada modulo se prueba de forma independiente. Los modulos se van integrando para determinar si la comunicacion entre los modulos es correcta. Una vez se integran todos los modulos y se conforma el sistema completo se determina si cimple con los requerimientos.
Mantenimiento
  • Esta fase comienza cuando el SW se libera oficialmente para los clientes.
  • Se reparan fallos que se presenten durante el uso por parte de los clientes. Se implementan requerimientos adicionales de los clientes con respecto a mejoras. Se mejoran atributos del SW como confiabilidad, desempeño, etc.
  • des de cada Mantenimiento se genera una nueva version del SW.
